3. 🛠️ SETUP GUIDE
--------------------------------------------------------------------------------

[Step 1: Create a Telegram Bot]
1. Open a chat with @BotFather on Telegram.
2. Issue the engine initialization sequence command: /newbot
3. Provide a standard descriptive title name (e.g., Lead Tracker Bot) and a 
   structurally unique handle username ending strictly in 'bot' 
   (e.g., MyLeadDataBot).
4. Copy the HTTP API Token generated by BotFather (formatted as 
   8833221126:AAGprr3hsHyQa...). Treat this token as a production security variable.

[Step 2: Set Up Google Sheets]
1. Create a fresh Google Spreadsheet.
2. Establish your relational header variables across the first row matching your 
   data schema requirements:
   - Column A: NAME
   - Column B: INSTA LINK
   - Column C: CONTACT NO.
   - Column D: STATUS
3. Navigate to the top options row menu bar, select Extensions, and enter the 
   Apps Script development ecosystem runtime environment.

[Step 3: Add and Configure Code]
1. Wipe clean any starter template boilerplate within your editor framework 
   workspace window.
2. Standardize your runtime variables by implementing the configuration block 
   template schema below into Code.js:

--- ENGINE CONFIGURATION SCRIPT START ---
const CONFIG = {
  TELEGRAM_TOKEN: "YOUR_TELEGRAM_BOT_TOKEN_HERE", 
  SHEET_NAME: "Sheet1"                            
};

function doPost(e) {
  try {
    const contents = JSON.parse(e.postData.contents);
    if (!contents.message || !contents.message.text) return;
    
    const chatId = contents.message.chat.id;
    const text = contents.message.text;
    
    if (text.toLowerCase() === "/start") {
      sendText(chatId, "👋 Bot is active!\n\nPlease send data using this exact comma format:\nName, Insta Link, Contact No, Status");
      return;
    }
    
    const dataArray = text.split(",").map(item => item.trim());
    if (dataArray.length < 4) {
      sendText(chatId, "❌ Formatting Error!\n\nPlease use the exact format:\nName, Insta Link, Contact No, Status");
      return;
    }
    
    const name = dataArray[0];
    const instaLink = dataArray[1];
    const contact = dataArray[2];
    const status = dataArray[3];
    
    const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheets()[0];
    sheet.appendRow([name, instaLink, contact, status]);
    
    sendText(chatId, "✅ Added '" + name + "' to the sheet successfully!");
  } catch(error) {
    Logger.log(error.toString());
  }
}

function sendText(chatId, text) {
  const url = "https://api.telegram.org/bot" + CONFIG.TELEGRAM_TOKEN + "/sendMessage";
  const payload = {
    "method": "post",
    "payload": {
      "chat_id": String(chatId),
      "text": text
    }
  };
  UrlFetchApp.fetch(url, payload);
}

function setWebhook() {
  const scriptUrl = ScriptApp.getService().getUrl();
  if (!scriptUrl || scriptUrl.indexOf("exec") === -1) {
    Logger.log("❌ Error: Deploy your script as a Web App first!");
    return;
  }
  const url = "https://api.telegram.org/bot" + CONFIG.TELEGRAM_TOKEN + "/setWebhook?url=" + scriptUrl;
  const response = UrlFetchApp.fetch(url);
  Logger.log("Result: " + response.getContentText());
}
--- ENGINE CONFIGURATION SCRIPT END ---

3. Paste your production-level TELEGRAM_TOKEN within the core CONFIG constants assignment line string.
4. Persist code modifications by clicking the Save interface platform icon.

[Step 4: Deploy as a Web App]
1. Locate the deployment options container array: Deploy -> New deployment.
2. Select the configuration setup gear component icon next to 'Select type' and initialize Web app.
3. Configure authentication bindings exactly as follows:
   - Execute as: Me (your-email@gmail.com)
   - Who has access: Anyone (Mandatory configuration block required to expose standard webhook listening routes to Telegram's dispatch servers).
4. Execute Deploy and confirm Google OAuth permissions to bind security allowances to the underlying scripting macro engine.

[Step 5: Bind the Webhook Connection]
1. From the primary script toolbar utility dropdown window, change the operational pointer tracking route target from 'doPost' to 'setWebhook'.
2. Initialize execution manually by hitting the Run button interface element.
3. Review the terminal execution frame outputs log. If a valid confirmation returns ({"ok":true,"result":true...}), the operational loop state is securely online.

5. 🔍 TROUBLESHOOTING & FAIL-SAFES
--------------------------------------------------------------------------------
* Bot fails to interact or return states on /start command execution:
  - Root Cause Analysis: Deployment permissions restriction settings block incoming JSON data streams.
  - Remediation Protocol: Re-evaluate deployment states container configuration (Deploy -> Manage Deployments). Ensure the access boundary layer property parameters are assigned explicitly to Anyone instead of tracking properties bound strictly to 'Only myself'.

* Webhook instantiation returns structural 404 network status errors:
  - Root Cause Analysis: The core runtime variable parsing step read an illegal token format assignment.
  - Remediation Protocol: Check character mappings under your TELEGRAM_TOKEN key assignment parameter matching your original source key structure within the @BotFather console context.

* Live modifications to scripting loops are not applying downstream:
  - Root Cause Analysis: Google cloud containers persist active routing nodes pointing directly to stale historical production compilation manifests.
  - Remediation Protocol: Forcing manifest updates requires an incremental version release step sequence: Open your script window -> Deploy -> Manage deployments -> Select the deployment asset Edit (Pencil representation tool) -> Toggle Version parameter to create a New version instance asset -> Commit changes via Deploy execution interface -> Re-trigger macro execution targeting setWebhook.
